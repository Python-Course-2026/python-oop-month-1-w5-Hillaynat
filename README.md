## 🛠 Справочник: Технологии и методы Python для работы с файлами

#### В этом документе разобраны все ключевые инструменты, которые мы использовали в лабораторных работах. Понимание этих основ превращает «написание кода» в «проектирование системы».

### 1. Контекстный менеджер (with ... as)

Что это: Специальная конструкция для безопасного управления внешними ресурсами.

Проблема: Файлы - это ресурсы ОС. Если открыть файл и забыть вызвать .close(), данные могут не сохраниться (остаться в буфере), а файл будет заблокирован для других программ.

Решение: Оператор with гарантирует закрытие файла автоматически сразу после выхода из блока кода, даже если внутри произошла критическая ошибка (Exception).

### 2. Работа со строками (Строковые методы)

Данные из файлов всегда приходят в виде строк (str). Чтобы с ними работать, мы использовали:

.strip(): Удаляет «невидимые» символы по краям (пробелы, табуляции и символ переноса строки \n).

Зачем: Без этого в конце данных часто остаются лишние переносы, которые создают пустые строки в JSON или ломают логику сравнения.

.split(separator): Разрезает строку на список по заданному разделителю.

Пример: "1,Alex,20".split(",") превращается в ['1', 'Alex', '20'].

.join(list): Собирает список строк в одну большую строку, вставляя разделитель между элементами.

Пример: "\n".join(['a', 'b']) создаст текст, где 'a' и 'b' на разных строках.

.startswith(string): Проверяет, начинается ли строка с определенных символов. Идеально подходит для фильтрации строк в конфигурационных файлах или логах.

### 3. Списки и Срезы (Slices)

Списки - это основной способ хранения данных в оперативной памяти.

Срезы (lines[1:]): Позволяют взять часть списка. Индекс до двоеточия - это старт (включительно), после - конец (не включая). [1:] означает: «пропусти первый элемент (заголовок) и возьми всё остальное».

Генераторы списков (List Comprehensions): Позволяют создавать новые списки в одну строку.

Синтаксис: [выражение for элемент in последовательность if условие].

Зачем: Это работает быстрее обычного цикла for и делает код лаконичнее.

### 4. Функция zip() и словари

Метафора: Застежка-молния.

Что делает: Берет несколько последовательностей (например, два списка) и объединяет их элементы попарно: первый с первым, второй со вторым.

Применение: Мы объединяем список имен столбцов (headers) и список значений (values). Затем функция dict() превращает эти пары в полноценный объект-словарь.

dict(zip(['name'], ['Alex'])) -> {'name': 'Alex'}

### 5. Продвинутые инструменты итерации

enumerate(sequence): Позволяет перебирать элементы списка, одновременно получая их индекс.

Зачем: Незаменимо, когда нужно изменить конкретную строку в списке (например, при патчинге конфига).

repr(): Функция, которая показывает «сырое» представление объекта.

Зачем: Помогает увидеть скрытые символы, такие как \n или \t, если вы не понимаете, почему файл читается некорректно.

### 6. Модуль json (Сериализация)

JSON (JavaScript Object Notation) - текстовый формат обмена данными, понятный и человеку, и машине.

json.dump(obj, file): "Сбрасывает" Python-объект в файл.

indent=4: Параметр, который добавляет пробелы и переносы строк, делая JSON "красивым" (Pretty Print).

ensure_ascii=False: По умолчанию JSON заменяет не-латинские символы на коды (например, \u0430). Этот флаг заставляет модуль сохранять русский текст как есть.

### 7. Обработка исключений (try...except)

Механизм, предотвращающий «падение» программы при предсказуемых ошибках.

Логика: Мы помещаем «опасный» код (открытие файла) в блок try. Если файл не найден, Python выбрасывает FileNotFoundError. Блок except перехватывает эту ошибку, позволяя нам записать сообщение в лог вместо того, чтобы программа закрылась с ошибкой.

### 8. Модуль os и pathlib

Работа с файловой системой требует кроссплатформенности (Windows использует \, а Linux /).

os.walk(): Рекурсивно проходит по дереву каталогов. На каждом шагу он выдает: текущий путь, список папок и список файлов.

os.path.join(): Соединяет части пути, используя правильный разделитель для текущей операционной системы.

### 9. Бинарные режимы и Сигнатуры

'rb' / 'wb': Режимы для работы с байтами (байтовыми строками вида b'\x00').

Магические байты (Signatures): Первые несколько байт файла, которые определяют его реальный тип. Например, у PNG это всегда \x89PNG. Проверка сигнатур надежнее, чем проверка расширения файла.

### 10. Кодировки и Память

UTF-8: Стандарт кодирования символов. Указывая encoding='utf-8', мы гарантируем, что программа одинаково прочитает русский текст на любой ОС.

Потоковое чтение: Использование for line in f: вместо f.read() позволяет обрабатывать файлы размером в несколько гигабайт, так как в памяти одновременно находится только одна строка текста.
